15/* RFC-6455 Section 5.2 Base Framing Protocol
16 * Payload length: 7 bits, 7+16 bits, or 7+64 bits
17 *
18 * The length of the "Payload data", in bytes: if 0-125, that is the
19 * payload length. If 126, the following 2 bytes interpreted as a
20 * 16-bit unsigned integer are the payload length. If 127, the
21 * following 8 bytes interpreted as a 64-bit unsigned integer (the
22 * most significant bit MUST be 0) are the payload length. Multibyte
23 * length quantities are expressed in network byte order. Note that
24 * in all cases, the minimal number of bytes MUST be used to encode
25 * the length, for example, the length of a 124-byte-long string
26 * can't be encoded as the sequence 126, 0, 124. The payload length
27 * is the length of the "Extension data" + the length of the
28 * "Application data". The length of the "Extension data" may be
29 * zero, in which case the payload length is the length of the
30 * "Application data".
31 */
