◆ connection_window

size_t aws_h1_connection::connection_window

The connection's current window size. We use this variable, instead of the existing aws_channel_slot.window_size, because that variable is not updated immediately, the channel uses a task to update it. Since we use the difference between current and desired window size when deciding how much to increment, we need the most up-to-date values possible.

◆ 

struct { ... } aws_h1_connection::read_buffer

All aws_io_messages arriving in the read direction are queued here before processing. This allows the connection to receive more data than the the current HTTP-stream might allow, and process the data later when HTTP-stream's window opens or the next stream begins.

The aws_io_message.copy_mark is used to track progress on partially processed messages. pending_bytes is the sum of all unprocessed bytes across all queued messages. capacity is the limit for how many unprocessed bytes we'd like in the queue.
