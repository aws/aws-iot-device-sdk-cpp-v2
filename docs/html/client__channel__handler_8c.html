bool  noRetry

This function registers a new outstanding request and returns the message identifier to use (or 0 on error). send_request will be called from request_timeout_task if everything succeed. Not called with error. on_complete will be called once the request completed, either either in success or error. noRetry is true for the packets will never be retried or offline queued.

QoS 0 publish and PINGREQ don't need a packet ID, but, we may still need a ID for publish QoS 0, For user to keep all data alive, an ID will help.
