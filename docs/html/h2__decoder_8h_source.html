12/* Decoder design goals:
13 * - Minimize state tracking and verification required by user.
14 * For example, we have _begin()/_i()/_end() callbacks when something happens N times.
15 * The _begin() and _end() callbacks tell the user when to transition states.
16 * Without them the user needs to be like, oh, I was doing X but now I'm doing Y,
17 * so I guess I need to end X and start Y.
18
19 * - A callback should result in 1 distinct action.
20 * For example, we have distinct callbacks for `on_ping()` and `on_ping_ack()`.
21 * We COULD have had just one `on_ping(bool ack)` callback, but since user must
22 * take two complete different actions based on the ACK, we opted for two callbacks.
23 */
24
