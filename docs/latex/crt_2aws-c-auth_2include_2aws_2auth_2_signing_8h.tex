\doxysubsubsection{\texorpdfstring{aws\_signing\_complete\_fn}{aws\_signing\_complete\_fn}}
{\footnotesize\ttfamily typedef void() aws\+\_\+signing\+\_\+complete\+\_\+fn(struct \mbox{\hyperlink{structaws__signing__result}{aws\+\_\+signing\+\_\+result}} $\ast$result, int \mbox{\hyperlink{channel__handler__test_8c_a7d16b1c68c87cec009d972e79abfba78}{error\+\_\+code}}, void $\ast$userdata)}

Gets called by the signing function when the signing is complete.

Note that result will be destroyed after this function returns, so either copy it, or do all necessary adjustments inside the callback.

When performing event or chunk signing, you will need to copy out the signature value in order to correctly configure the signable that wraps the event or chunk you want signed next. The signature is found in the \char`\"{}signature\char`\"{} property on the signing result. This value must be added as the \char`\"{}previous-\/signature\char`\"{} property on the next signable.
