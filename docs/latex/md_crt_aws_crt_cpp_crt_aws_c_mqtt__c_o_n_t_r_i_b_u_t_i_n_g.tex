We welcome you to use the Git\+Hub issue tracker to report bugs or suggest features.

When filing an issue, please check \href{https://github.com/awslabs/aws-c-mqtt/issues}{\texttt{ existing open}}, or \href{https://github.com/awslabs/aws-c-mqtt/issues?utf8=\%E2\%9C\%93&q=is\%3Aissue\%20is\%3Aclosed\%20}{\texttt{ recently closed}}, issues to make sure somebody else hasn\textquotesingle{}t already reported the issue. Please try to include as much information as you can. Details like these are incredibly useful\+:


\begin{DoxyItemize}
\item A reproducible test case or series of steps
\item The version of our code being used
\item Any modifications you\textquotesingle{}ve made relevant to the bug
\item Anything unusual about your environment or deployment
\end{DoxyItemize}\hypertarget{md_crt_aws_crt_cpp_crt_aws_c_mqtt__c_o_n_t_r_i_b_u_t_i_n_g_autotoc_md294}{}\doxysection{Contributing via Pull Requests}\label{md_crt_aws_crt_cpp_crt_aws_c_mqtt__c_o_n_t_r_i_b_u_t_i_n_g_autotoc_md294}
Contributions via pull requests are much appreciated. Before sending us a pull request, please ensure that\+:


\begin{DoxyEnumerate}
\item You are working against the latest source on the {\itshape main} branch.
\item You check existing open, and recently merged, pull requests to make sure someone else hasn\textquotesingle{}t addressed the problem already.
\item You open an issue to discuss any significant work -\/ we would hate for your time to be wasted.
\end{DoxyEnumerate}

To send us a pull request, please\+:


\begin{DoxyEnumerate}
\item Fork the repository.
\item Modify the source; please focus on the specific change you are contributing. If you also reformat all the code, it will be hard for us to focus on your change.
\item Ensure local tests pass.
\item Commit to your fork using clear commit messages.
\item Send us a pull request, answering any default questions in the pull request interface.
\item Pay attention to any automated CI failures reported in the pull request, and stay involved in the conversation.
\end{DoxyEnumerate}
