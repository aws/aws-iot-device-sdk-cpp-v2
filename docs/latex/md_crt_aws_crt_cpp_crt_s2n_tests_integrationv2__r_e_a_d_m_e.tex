If you are testing a new feature you need to determine how to use that feature with all supported providers.\hypertarget{md_crt_aws_crt_cpp_crt_s2n_tests_integrationv2__r_e_a_d_m_e_autotoc_md554}{}\doxysubsection{s2nd / s2nc}\label{md_crt_aws_crt_cpp_crt_s2n_tests_integrationv2__r_e_a_d_m_e_autotoc_md554}
You may have to add a command line flag to \mbox{\hyperlink{s2nc_8c}{s2nc.\+c}} or \mbox{\hyperlink{s2nd_8c}{s2nd.\+c}} for your feature. Use long options if possible, and use the same option name in both \mbox{\hyperlink{s2nc_8c}{s2nc.\+c}} and \mbox{\hyperlink{s2nd_8c}{s2nd.\+c}}. If you are able to use an option name similar to the Open\+SSL name, please do. This reduces complexity across TLS providers.

An example of similar naming is \textquotesingle{}-\/reconnect\textquotesingle{} in Open\+SSL and \textquotesingle{}-\/r\textquotesingle{} in S2N. Both have a hardcoded value of 5 reconnects. The point is to remove logic from the test, and make the providers act as similar as possible.\hypertarget{md_crt_aws_crt_cpp_crt_s2n_tests_integrationv2__r_e_a_d_m_e_autotoc_md555}{}\doxysubsection{Control the provider from the test}\label{md_crt_aws_crt_cpp_crt_s2n_tests_integrationv2__r_e_a_d_m_e_autotoc_md555}
If you are testing a feature which is similar across all TLS providers, add an option to the Provider\+Options object in {\ttfamily \mbox{\hyperlink{common_8py}{common.\+py}}}. If this feature is specific to a single provider, you can use the {\ttfamily extra\+\_\+flags} option.

For example, to test the session resumption feature we need to tell various clients and servers to resume a session multiple times. To do this we added the \textquotesingle{}reconnect\textquotesingle{} and \textquotesingle{}reconnects\+\_\+before\+\_\+exit\textquotesingle{} options to the Provider\+Options object. But with the dynamic threshold feature we simply pass the \textquotesingle{}-\/D\textquotesingle{} argument as an {\ttfamily extra\+\_\+flag} to s2n.

In each provider that supports your feature you need to check if the flag is set, and create a command line option for that particular provider. You can also add logic checks, e.\+g with client authentication the client must have a certificate to send. Otherwise the test will fail.\hypertarget{md_crt_aws_crt_cpp_crt_s2n_tests_integrationv2__r_e_a_d_m_e_autotoc_md556}{}\doxysubsection{Fine-\/tune how input data is sent in the tests}\label{md_crt_aws_crt_cpp_crt_s2n_tests_integrationv2__r_e_a_d_m_e_autotoc_md556}
There are several optional arguments in the managed\+\_\+process function. Use these arguments to have a greater level of control over how peers send test data. The argument send\+\_\+marker can be used in parallel with the data\+\_\+to\+\_\+send Provider options. If both of these arguments are lists, the process will only write a data\+\_\+to\+\_\+send element when it reads a corresponding send\+\_\+marker element from the process\textquotesingle{}s stdout or stderr. Additionally, the managed\+\_\+process argument close\+\_\+marker can be used to control when the process should shut down.
