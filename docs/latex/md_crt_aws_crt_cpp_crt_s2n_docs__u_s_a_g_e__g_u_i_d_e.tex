When s2n is used in non-\/blocking mode, this set of functions allows user to move execution of CPU-\/heavy private key operations out of the main event loop, preventing {\bfseries{s2n\+\_\+negotiate}} blocking the loop for a few milliseconds each time the private key operation needs to be performed.

To enable asynchronous private key operations user needs to provide a callback function {\bfseries{s2n\+\_\+async\+\_\+pkey\+\_\+fn}} to {\bfseries{s2n\+\_\+config\+\_\+set\+\_\+async\+\_\+pkey\+\_\+callback}} call. This function will be executed during {\bfseries{s2n\+\_\+negotiate}} call every time an operation on private key needs to be performed. The argument {\bfseries{op}} represents the operation to perform. From the callback the user can spawn the thread to perform {\bfseries{op}} through {\bfseries{s2n\+\_\+async\+\_\+pkey\+\_\+op\+\_\+perform}} call and immediately return {\bfseries{S2\+N\+\_\+\+SUCCESS}} from the function without waiting for thread to complete. The {\bfseries{s2n\+\_\+negotiate}} will return {\bfseries{S2\+N\+\_\+\+FAILURE}} with {\bfseries{S2\+N\+\_\+\+ERR\+\_\+\+T\+\_\+\+BLOCKED}} error type and {\bfseries{s2n\+\_\+blocked\+\_\+status}} {\bfseries{S2\+N\+\_\+\+BLOCKED\+\_\+\+ON\+\_\+\+APPLICATION\+\_\+\+INPUT}}, and will keep giving the same error until the {\bfseries{op}} is performed and applied to the connection through {\bfseries{s2n\+\_\+async\+\_\+pkey\+\_\+op\+\_\+apply}} call.

Note, it is not safe to call multiple functions on the same {\bfseries{conn}} or {\bfseries{op}} objects from 2 different threads at the same time. Doing so will produce undefined behaviour. However it is safe to have a call to function involving only {\bfseries{conn}} at the same time with a call to function involving only {\bfseries{op}}, as those 2 objects are not coupled with each other. It is also safe to free {\bfseries{conn}} or {\bfseries{op}} at any moment with respective function calls, with the only exception that {\bfseries{conn}} cannot be freed inside the {\bfseries{s2n\+\_\+async\+\_\+pkey\+\_\+fn}} callback.

{\bfseries{s2n\+\_\+async\+\_\+pkey\+\_\+fn}} is invoked every time some action involving private key is required during {\bfseries{s2n\+\_\+negotiate}}. The {\bfseries{conn}} provides a pointer to the connection which triggered the callback, the {\bfseries{op}} is a pointer to an operation to be performed. The callback takes the ownership of {\bfseries{op}} object and is responsible for freeing the memory for it.

{\bfseries{s2n\+\_\+config\+\_\+set\+\_\+async\+\_\+pkey\+\_\+callback}} sets up the callback to invoke for asynchronous private key operations and enables asynchronous mode.

{\bfseries{s2n\+\_\+async\+\_\+pkey\+\_\+op\+\_\+perform}} performs the {\bfseries{op}} allowing it to be used to resume the handshake through {\bfseries{s2n\+\_\+async\+\_\+pkey\+\_\+op\+\_\+apply}} call. This function can be called only once and any subsequent calls will produce a failure. It is safe to call from a different thread, as long as no other thread is operating on {\bfseries{op}}.

{\bfseries{s2n\+\_\+async\+\_\+pkey\+\_\+op\+\_\+apply}} applies the performed {\bfseries{op}} to {\bfseries{conn}} allowing for the next call to {\bfseries{s2n\+\_\+negotiate}} to proceed through handshake. The function will fail if it is called from {\bfseries{s2n\+\_\+async\+\_\+pkey\+\_\+fn}} callback, or if {\bfseries{op}} was not performed through {\bfseries{s2n\+\_\+async\+\_\+pkey\+\_\+op\+\_\+perform}} call, or if provided {\bfseries{conn}} is different from the original {\bfseries{conn}} which initiated callback for this {\bfseries{op}}. The function will succeed only once and any subsequent call will result in failure for the same {\bfseries{op}}.

{\bfseries{s2n\+\_\+async\+\_\+pkey\+\_\+op\+\_\+free}} frees the memory for {\bfseries{op}}. {\bfseries{s2n\+\_\+shutdown}} attempts a closure at the TLS layer. It does not close the underlying transport. The call may block in either direction. Unlike other TLS implementations, {\bfseries{s2n\+\_\+shutdown}} attempts a graceful shutdown by default. It will not return with success unless a close\+\_\+notify alert is successfully sent and received. As a result, {\bfseries{s2n\+\_\+shutdown}} may fail when interacting with a non-\/conformant TLS implementation. Once {\bfseries{s2n\+\_\+shutdown}} is complete\+:
\begin{DoxyItemize}
\item The \mbox{\hyperlink{structs2n__connection}{s2n\+\_\+connection}} handle cannot be used for reading for writing.
\item The underlying transport can be closed. Most likely via {\ttfamily \mbox{\hyperlink{new__destroy_8c_a6cc331334d030ff39790e12d10ef25c1}{close()}}}.
\item The \mbox{\hyperlink{structs2n__connection}{s2n\+\_\+connection}} handle can be freed via \href{\#s2n\\_connection\\_free}{\texttt{ s2n\+\_\+connection\+\_\+free}} or reused via \href{\#s2n\\_connection\\_wipe}{\texttt{ s2n\+\_\+connection\+\_\+wipe}}

{\bfseries{s2n\+\_\+mem\+\_\+set\+\_\+callbacks}} allows the caller to over-\/ride s2n\textquotesingle{}s internal memory handling functions. To work correctly, {\bfseries{s2n\+\_\+mem\+\_\+set\+\_\+callbacks}} must be called before {\bfseries{s2n\+\_\+init}}. {\bfseries{s2n\+\_\+mem\+\_\+init\+\_\+callback}} should be a function that will be called when s2n is initialized. {\bfseries{s2n\+\_\+mem\+\_\+cleanup\+\_\+callback}} will be called when {\bfseries{s2n\+\_\+cleanup}} is executed. {\bfseries{s2n\+\_\+mem\+\_\+malloc\+\_\+callback}} should be a function that can allocate at least {\bfseries{requested}} bytes of memory and store the location of that memory in $\ast$$\ast$$\ast$ptr$\ast$$\ast$, and the size of the allocated data in $\ast$$\ast$$\ast$allocated$\ast$$\ast$. The function may choose to allocate more memory than was requested. s2n will consider all allocated memory available for use, and will attempt to free all allocated memory when able. {\bfseries{s2n\+\_\+mem\+\_\+free\+\_\+callback}} should be a function that can free memory.\hypertarget{md_crt_aws_crt_cpp_crt_s2n_docs__u_s_a_g_e__g_u_i_d_e_autotoc_md469}{}\doxysection{Examples}\label{md_crt_aws_crt_cpp_crt_s2n_docs__u_s_a_g_e__g_u_i_d_e_autotoc_md469}
